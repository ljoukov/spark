# PDF Transcription Benchmark

This benchmark compares end-to-end transcription quality for the Hamilton 2017 PDF using multiple extraction approaches.

## What It Tests

- `PDF + gemini-flash-latest`
- `PDF + gemini-2.5-pro`
- `PDF as images + chatgpt-gpt-5.3-codex`
- `PDF as images + gpt-5.2`

Each approach is evaluated across:

- extraction mode: `bulk` (H1/H2/H3 in one call) vs `individual` (one call per problem)
- coordinate mode: `norm` (`bboxNorm`) vs `int1000` (`bbox1000`)

The benchmark runs all configurations in parallel, produces per-config artifacts, and then runs two judges:

- `gemini-2.5-pro`
- `chatgpt-gpt-5.3-codex`

## How To Run

From repo root:

```bash
bun --cwd=eval run bench:pdf-transcription
```

Optional custom source PDF:

```bash
bun --cwd=eval run bench:pdf-transcription -- --source-pdf=/absolute/path/to/file.pdf
```

## Outputs

- Detailed run JSON is written under `eval/src/benchmarks/pdf-transcription/runs/<run-id>/benchmark-results.json`.
- Per-config copied artifacts are written under `eval/src/benchmarks/pdf-transcription/output/<config-name>/`.
- `RESULTS.md` in this directory is auto-generated by each benchmark run.
